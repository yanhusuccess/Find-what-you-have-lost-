# 快速开始指南

## 📌 系统要求

- Python 3.7 或更高版本
- 现代浏览器（Chrome、Firefox、Edge、Safari等）
- 至少200MB磁盘空间

## 🚀 三步启动

### Windows用户

1. **双击运行启动脚本**
   ```
   双击 run.bat 文件
   ```
   脚本会自动：
   - 创建虚拟环境
   - 安装所需依赖
   - 启动Web服务器

2. **访问网站**
   - 打开浏览器访问：http://localhost:5000
   - 管理后台：http://localhost:5000/admin

3. **开始使用**
   - 注册新账号或使用默认管理员账号登录
   - 默认管理员：用户名 `admin`，密码 `admin123`

### Linux/Mac用户

1. **运行启动脚本**
   ```bash
   chmod +x run.sh
   ./run.sh
   ```

2. **访问网站**（同上）

## 📖 手动启动（可选）

### 1. 安装依赖
```bash
# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
# Windows:
venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate

# 安装依赖包
pip install -r requirements.txt
```

### 2. 运行应用
```bash
python app.py
```

### 3. 访问应用
浏览器打开：http://localhost:5000

## 🎯 主要功能体验

### 1. 注册和登录
- 访问首页点击"注册"
- 填写用户名、邮箱、密码
- 登录系统

### 2. 发布失物信息
- 登录后点击"发布失物"
- 填写物品信息（名称、描述、地点、日期等）
- 可选择上传物品图片
- 提交发布

### 3. 发布拾物信息
- 登录后点击"发布拾物"
- 填写拾到物品的详细信息
- 上传物品图片
- 提交发布

### 4. 搜索物品
- 进入"寻物启事"或"拾物认领"页面
- 使用搜索框输入关键词
- 选择物品类别进行筛选
- 浏览搜索结果

### 5. 查看详情和互动
- 点击任意物品卡片查看详情
- 可以：
  - 发表评论
  - 发送私信给发布者
  - 更新物品状态（自己发布的）

### 6. 消息系统
- 点击导航栏"消息"图标
- 查看收到的消息和发送的消息
- 红色徽章显示未读消息数量
- 点击消息查看详情并回复

### 7. 个人中心
- 点击用户名下拉菜单
- 选择"个人中心"
- 查看和管理自己发布的失物和拾物

### 8. 数据统计
- 点击"数据统计"
- 查看各类图表：
  - 失物/拾物类别分布（饼图）
  - 状态统计（柱状图）
  - 失物与拾物对比（柱状图）
  - 总体统计数据

### 9. 管理后台（管理员）
- 使用管理员账号登录
- 访问：http://localhost:5000/admin
- 可管理：
  - 用户
  - 失物信息
  - 拾物信息
  - 评论
  - 消息

## ⚙️ 常见问题

### Q1: 如何修改端口？
在 `app.py` 文件最后一行修改：
```python
app.run(debug=True, host='0.0.0.0', port=5000)  # 修改port参数
```

### Q2: 如何修改上传文件大小限制？
在 `app.py` 中修改：
```python
app.config['MAX_CONTENT_LENGTH'] = 16 * 1024 * 1024  # 16MB，可调整
```

### Q3: 如何备份数据？
复制 `lostfound.db` 文件即可备份所有数据。

### Q4: 如何重置数据库？
删除 `lostfound.db` 文件，重新运行 `python app.py` 会自动创建新数据库。

### Q5: 上传的图片保存在哪里？
所有上传的图片保存在 `static/uploads/` 目录中。

### Q6: 如何停止服务器？
在终端窗口按 `Ctrl + C` 即可停止。

## 🛡️ 安全提示

1. **生产环境部署**：
   - 务必修改 `SECRET_KEY`
   - 修改默认管理员密码
   - 使用HTTPS
   - 配置防火墙

2. **数据备份**：
   - 定期备份 `lostfound.db` 数据库文件
   - 备份 `static/uploads/` 图片目录

3. **密码安全**：
   - 使用强密码
   - 不要共享账号
